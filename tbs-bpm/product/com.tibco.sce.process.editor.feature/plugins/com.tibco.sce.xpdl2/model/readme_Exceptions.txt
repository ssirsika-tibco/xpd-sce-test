All extension ‘element’ (not attribute) declarations in document root must be unbounded (upperbound=-1). 
Otherwise you will get the following exception (and possibly others)…

The reason to have it unbounded can be explained as below:

- The ##other is a multi-instance featuremap (a list of entries) EMF expects that all contents is part of a multi-instance sequence.

- Therefore when it tries to unlink the extension elements from the feature map it is expecting to remove a list from a list and finds a single instance and blows
