How To Contribute Sample Business Object Model Fragments

1. Create the fragments:

  1. For each sample fragment create a BOM model and create the "fragment" (the contents of the Model will be converted to a fragment). The Model's name and description will be used as the fragment's name and description respectively.
  2. Externalize the model: open the BOM model in the UML editor and use the menu option UML Editor->Externalize->Labels. This will create a properties file at the same location as the BOM model. Notice that Model description will not be externalized and neither will any Association names. The properties files can be updated with Association labels manually if localization of Associations is required (following the key structure in the properties file). To externalize the description of the Model add a key with the prefix "_shortdesc" (alongside the "_label" entry) for the model.

2. Update this plug-in with the sample fragments:

  1. Copy the created BOM models and their corresponding properties files into the fragments folder in this plug-in (or any other folder in the plug-in, but this folder is recommended for clarity). NOTE: Sub-folders should be created for every sample fragment category.
  2. Extend private extension point com.tibco.xpd.bom.fragments.bomFragment for each sample category. The extension will require the name and description of the category - this will appear in the fragments view - and also the location of the category contribution. The extension will read the contents of the folder and create fragments for each .bom file found.
  3. Update the Fragments contributor's version number to indicate, to the fragments controller, that new sample fragments are available (see BOMFragmentsContributor.getFragmentVersion).

3. Localization of the fragments:

  1. The standard Eclipse localization method of properties should be followed, ie create language specific files such as fragment1_fr.properties, etc. Plug-in fragments are also supported by the fragment localization system.

NOTE: Don't forget to include the fragments folder in the build.
